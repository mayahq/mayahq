# Admin - Maya's Activity Feed

This section of the admin panel provides a comprehensive view and management interface for all items generated by Maya or other integrated systems, intended for review, approval, and further processing.

## Overview

The feed displays various types of content, including text messages generated by the Mood Engine, images generated via ComfyUI (either directly or through the Mood Engine), and image series.

## Key Features

- **Unified View**: Displays diverse item types (text, single images, image series masters) in a chronological or filterable list.
- **Filtering**: Users can filter feed items by:
    - Status (e.g., `pending_review`, `approved`, `prompt_generated`, `image_generated_pending_review`, `series_generated`)
    - Item Type (e.g., `text_mood_engine`, `image_mood_engine`, `image_comfyui`, `image_studio_series_master`, `image_series_variation`)
    - Source System (e.g., `MoodEngine`, `ComfyUI`, `SeriesGenerator`)
    - Date Range
- **Content Display**:
    - Text items show generated text and associated mood.
    - Image items display the generated image, the prompt used, and associated context (like mood).
    - For images being processed by ComfyUI (status `prompt_generated`), a placeholder indicates that generation is in progress.
- **Actions (Role-Based)**:
    - **Approve**: Marks an item as approved, potentially making it eligible for further actions (e.g., social media posting, series generation).
    - **Reject**: Marks an item as rejected, with an option to add review notes.
    - **Edit Content**: Allows modification of an item's `content_data` (e.g., editing text, image prompts) while it's in a reviewable state.
    - **Like/Unlike**: Users can like or unlike individual feed items (both master items and variations within a series). Liked items can be targeted by other processes (e.g., n8n for publishing).
- **Series Gallery**: For master items that represent an image series (`status: 'series_generated'`), a "View Series" button opens a modal displaying all individual variation images associated with that master item.
    - Each variation within the modal can also be individually liked.
- **Image Lightbox**: Clicking on an image in the feed or series gallery opens a larger lightbox view.
- **Comments**: Users can add and view comments on feed items.

## Technical Details

- **Component**: `website/src/app/admin/feed/page.tsx`
- **Data Fetching**: Retrieves feed items from the `/api/v1/feed/items` endpoint (provided by the `memory-worker` service).
- **Like/Unlike API**: Uses `/api/v1/feed/items/[itemId]/like` and `/api/v1/feed/items/[itemId]/unlike` endpoints (defined in `website/src/app/api/...`).
- **Series Variation Fetching**: The Series Gallery modal fetches individual `image_series_variation` items by querying `feed_items` where `parent_feed_item_id` matches the master item's ID.

## Data Flow for Image Series

1.  **Master Item Creation**: A master item (e.g., `image_studio_series_master` or `image_mood_engine` with status `prompt_generated`) is created.
2.  **Series Generation Trigger**: An external process (e.g., n8n calling the `series-generator` service) is triggered, referencing the master item's ID.
3.  **Variation Creation (`series-generator` service)**:
    - Generates multiple image variations.
    - For each variation, it uploads the image to S3.
    - **Crucially, it inserts a new row into the `feed_items` table** for each variation with:
        - `item_type: 'image_series_variation'`
        - `parent_feed_item_id`: set to the ID of the master item.
        - `content_data` containing the variation's image URL, prompt, etc.
        - Appropriate `status` (e.g., `approved` or `image_generated_pending_review`).
    - It also updates the master item's `status` to `series_generated` and can optionally populate a JSONB array (`generated_series_data`) on the master item with quick-access details of the variations (though the modal now relies on fetching the full variation items).
4.  **Display in Modal (`FeedPage.tsx`)**:
    - When "View Series" is clicked on a master item, `fetchSeriesVariationFeedItems` queries for all `feed_items` whose `parent_feed_item_id` matches the master item.
    - These full variation `FeedItem` objects are then displayed in the modal, allowing individual interaction (like liking).

## Future Considerations / Recreate Feature (Planned)

- A "Recreate" button will be added to master images and individual series variations.
- This will call a new endpoint in the `series-generator` (e.g., `/recreate-as-variation`).
- The endpoint will generate a new image based on the source item's prompt and create a new `image_series_variation` feed item, linking it to the relevant master item via `parent_feed_item_id`.
